1 RAILtest Changelist {#rail_test_changelist}
2 ============
3 
4 ### RAIL 2.10.x:
5 
6 * __605067:__ The SL_RAIL_TEST_PER warning has been removed from RAILtest when
7  a PER pin is not configured for use.
8 
9 ### RAIL 2.9.x:
10 * __381485:__ Added `configChannelHoppingMulti` command for pre-configuring
11  channel hopping multi-sense mode parameters prior to issuing
12  `configRxChannelHopping` command. Also augmented `configRxDutyCycle`
13  command to allow parameterization of this mode.
14 * __402203:__ Augmented `configChannelHoppingOptions` command to allow
15  specifying RSSI threshold when enabling that option.
16  Augmented `configRxDutyCycle` command to allow specifying
17  options in general including the RSSI threshold.
18 * __427841:__ Updated the Command Line Interface (CLI) help display format.
19 * __488425:__ CLI commands are now displayed back to the user using the
20  same capitalization as they are entered (e.g. entering command
21  `configTXoptions` will display `configTXoptions` and not `configTxOptions`).
22 * __454890:__ Restructured RAILtest to use a common main.c template with a
23  software component-based architecture to select software features.
24 * __487098:__ Disabled graphics, buttons, LEDs and PER test GPIO pin by default.
25 * __484333:__ Standardized spelling of `RAILtest` for consistency.
26  * Updated bootup message example:
27  `{{(reset)}{App:RAILtest}{Built:Jul 19 2019 13:36:24}}`
28 * __633846:__ Altered CLI input of command `setDebugSignal`.
29 * __630457:__ Fixed an issue where the RAIL PTI component would automatically
30  reserve hardware pins when no pins were specified for use.
31 
32 ### RAIL 2.8.x:
33 * __344361:__ Added new command `getCteSampleRate` to retrieve the actual ADC
34  sample rate used to capture the CTE.
35 * __369727:__ Added new commands `setTxTimePos` and `setRxTimePos` to
36  designate the time position of displayed packet timestamps among
37  the RAIL_PacketTimePosition_t choices. In txEnd messages, now also
38  display the lastTxStart time, and fixed a bug with the lastTxStatus
39  display.
40 * __406293:__ Fixed an issue where a memory pointer could be discarded from
41  the event queue on queue overflow and never released. This can occur if the
42  memory buffer pool is greater than the event queue size.
43 * __415017:__ Updated the RAILTest bootup message to incorporate the `reset`
44  command, the application name and the date and time of the RAILTest build.
45  * Updated bootup message example:
46  `{{(reset)}{App:RAIL Test App}{Built:Jul 19 2019 13:36:24}}`
47 * __406080:__ Added new commands `configRfSenseWakeupPhy` and
48  `setRfSenseTxPayload` to support waking up chips that support RFSENSE
49  Selective(OOK) mode. Updated `sleep` and `rfsense` commands to allow setting
50  the syncword length and syncword for EFR32XG22 chips which support RFSENSE
51  Selective(OOK) mode. The `tx` command will always overwrite the TX FIFO with
52  a default payload, hence `setRfSenseTxPayload` must be used with
53  `fifoModeTestOptions` set to use `txFifoManual` to prevent the FIFO being
54  overwritten.
55 * __460516:__ Fixed minor railtest problem where `setConfigIndex` might not
56  restart receiver listening like `setChannel` would if the receiver was on
57  when the command was issued.
58 
59 ### RAIL 2.7.x:
60 * __368510:__ Fixed an issue if receive was entered because of a transmit state
61  transition where we would not change the channel when calling setChannel and
62  would ignore calls to `rx 1` to enter normal receive mode.
63 * __394431:__ Fixed an issue where the configured antenna diversity
64  settings in HAL config would not be applied to the radio at startup.
65 
66 ### RAIL 2.6.x:
67 * Fixed `setMultiTimer` which was ignoring its mode argument; `abs` mode
68  now works.
69 * The commands `setRxOptions` and `configTxOptions` can now be called without
70  any parameters to print the current state of these options.
71 * RAILTest now uses its config-changed callback for both external and internal
72  radio configs to ensure that the appropriate PA level is always selected.
73 * Added 802.15.4 `setDataReqLatency` command for testing purposes.
74 * Fixed some asynchronous event print outs related to Z-Wave beam frame
75  reception since they could corrupt the output.
76 * Added `getChannelHoppingRssi` command to read the RSSI when in channel hopping
77  mode on each available channel.
78 
79 ### RAIL 2.5.x:
80 * Added the following multitimer commands.
81  * `enableMultiTimer`, `setMultiTimer`, `multiTimerCancel`,
82  `printMultiTimerStats`
83 * Added `printRxErrors` command to enable/disable printing of unsuccessfully
84  received packets.
85 * Added the ability to save command scripts to flash and run them from flash.
86  Renamed command `startScript` to `runScript`. Added command `clearScript`.
87 
88 ### RAIL 2.4.x:
89 * Added the RAM Modem Reconfiguration commands. These commands allow the
90  dynamic configuration of the modem via a set of studio scripted commands.
91 
92 ### RAIL 2.0.x:
93 * `setChannel` no longer requires the radio be idle.
94 
95 ### RAIL 1.6.x:
96 * simple `tx` command transmits with defaults (i.e. don't wait for ack,
97  send CRC, use sync word 0)
98 * configTxOptions now controls 3 options
99  * BIT0: Wait for ack
100  * BIT1: Send CRC
101  * BIT2: Sync Word ID
102 * setRxOptions now controls 3 options
103  * BIT0: Store CRC
104  * BIT1: Ignore CRC errors
105  * BIT2: Enable dual sync

137 ### RAIL 1.2.1:
138 * Added `setPaCtune` which can configure custom PA capacitor tuning values for

140 * Fixed RSSI slope requiring a 4dB power change for 1dB change in the received

143 ### RAIL 1.2.0:
144 * Added [bit error rate test](#bit-error-rate-testing) commands -
145  `setBerConfig`, `berRx`, and `berStatus`.
146 * Added [packet error rate test](#packet-error-rate-testing) commands -
147  `perRx`, `perStatus`.
148 * Added the ability to set the RAIL timebase in microseconds - `setTime`.
149 * RxOverflow counter now only counts number of overflows, instead of
150  mistakenly counting some non-overflow events.
151 * Removed the RfUs output parameter from the 'sleep' command due to it being
152  incorrect.
153 
154 ### RAIL 1.1 - May 15, 2016:
155 * [Automatic state transitions](#automatic-state-transitions) and timings added
156  via `setRxTransitions`, `setTxTransitions`, and `setTimings`.
157 * Ability to selectively enable the address filter on certain frame types
158  added through `addressFilterByFrame` command.
159 * Independent status booleans were removed in favor of a single `AppMode`
160  status variable, which keeps track of the state of RAILtest.
161 * More validation is done to ensure that certain RAIL commands are only run in
162  the correct AppMode and radio state. The radio must be IDLE for `setChannel`,
163  `freqOverride`, `setPower`, and `setCtune` to succeed.
164 * `setTestMode` was removed in favor of individual commands for
165  `setTxUnderflow`, `setRxOverflow`, and `setCalibrations`
166 * `toggleTone` became `setTxTone`, and `toggleStream` became `setTxStream`.
167 * Removed command `getTicks`, since the RAIL_GetTime() function made the
168  timer used unnecessary.
169 * Removed command `txTimeReport`, since it was exposing implemenation-specific
170  behavior encapsulated by RAIL.
171 
172 ### Beta 1 - November 16, 2015:
173 * New library features for RF Energy Sensing exposed via new `rfSense` and
174  `sleep` commands.
175 * New RAIL timer functionality exposed via the `setTimer`, `timerCancel`, and
176  `printTimerStats` commands.
177 * New `printDataRates` command to get information about the current PHY's data
178  rates.
179 * Documentation clean up.
180 
181 ### Alpha 2 - September 14, 2015:
182 * Commands for filtering of receive packets based on defined address fields
183  and compare values.
184 * Scheduling of packet transmission at an absolute time or after a defined
185  delay.
186 * Packet transmission conditional on Clear Channel Assessment (CSMA, LBT).
187 * The ability to read RSSI manually.
188 * Capability to switch between multiple PHY configurations at runtime.
189 * Revised help text (more uniform parameter descriptions).
